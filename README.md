# Учебный Django-блог

Учебный проект для студентов, которые начинают изучать Django. Проект показывает базовую структуру сайта-блога без Django REST Framework: обычные маршруты, function-based views, формы Django, шаблоны, статические файлы и работу с пользователями.

## Возможности

- регистрация пользователей;
- вход и выход из аккаунта;
- редактирование профиля пользователя;
- расширенная модель пользователя с полями:
  - телефон;
  - город;
  - аватар;
- создание постов;
- выбор категории при создании или редактировании поста;
- редактирование и удаление своих постов;
- просмотр чужих постов;
- просмотр всех постов конкретного пользователя;
- отдельная страница со своими постами;
- лайки постов;
- удаление чужих постов администратором;
- добавление до трех изображений к одному посту;
- поиск постов по заголовку и тексту;
- фильтрация по автору, категории и тегу;
- сортировка по дате, заголовку и количеству лайков.

## Технологии

- Python;
- Django 5.2;
- SQLite;
- HTML-шаблоны Django;
- CSS;
- простой JavaScript.

Проект специально не использует DRF, сложный frontend, webpack, Bootstrap и другие дополнительные инструменты, чтобы код оставался понятным для начинающих.

## Структура проекта

```text
.
├── blog/                 # Приложение блога: посты, категории, теги, лайки
├── config/               # Настройки Django-проекта
├── static/
│   ├── css/style.css     # Все основные стили проекта
│   └── js/main.js        # Простые JavaScript-примеры
├── templates/
│   ├── base.html         # Базовый шаблон
│   ├── blog/             # Шаблоны приложения blog
│   └── users/            # Шаблоны приложения users
├── users/                # Приложение пользователей
├── manage.py
└── requirements.txt
```

## Установка

Создайте и активируйте виртуальное окружение:

```bash
python3 -m venv .venv
source .venv/bin/activate
```

Установите зависимости:

```bash
pip install -r requirements.txt
```

## Миграции

Примените миграции:

```bash
python manage.py migrate
```

Если модели были изменены, сначала создайте новые миграции:

```bash
python manage.py makemigrations
python manage.py migrate
```

## Создание администратора

Для доступа в админку создайте суперпользователя:

```bash
python manage.py createsuperuser
```

Админка доступна по адресу:

```text
http://127.0.0.1:8000/admin/
```

## Запуск проекта

Запустите локальный сервер:

```bash
python manage.py runserver
```

Откройте сайт в браузере:

```text
http://127.0.0.1:8000/
```

## Основные маршруты

```text
/                         # список постов
/posts/my/                # мои посты
/posts/<id>/              # страница одного поста
/posts/create/            # создание поста
/posts/<id>/edit/         # редактирование поста
/posts/<id>/delete/       # удаление поста
/posts/<id>/like/         # лайк или снятие лайка
/users/<id>/posts/        # посты выбранного пользователя
/users/register/          # регистрация
/users/login/             # вход
/users/logout/            # выход
/users/profile/           # профиль пользователя
```

## Учебные особенности проекта

В проекте намеренно используется простой подход:

- views написаны как функции, а не как классы;
- формы создаются через `forms.ModelForm`;
- у каждого приложения есть свой `urls.py`;
- общий шаблон `base.html` используется как родительский шаблон;
- стили вынесены в `static/css/style.css`;
- JavaScript вынесен в `static/js/main.js`;
- категории сделаны через `ForeignKey`: у поста одна категория, у категории много постов;
- теги сделаны через `ManyToManyField`: у поста может быть несколько тегов;
- изображения поста хранятся в трех полях `ImageField`, чтобы не усложнять проект дополнительной моделью вложений;
- в коде оставлено много комментариев, объясняющих основные действия Django.

## Проверка проекта

Проверить настройки Django можно командой:

```bash
python manage.py check
```

Если команда завершается сообщением `System check identified no issues`, базовая конфигурация проекта корректна.
